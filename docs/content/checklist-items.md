## チェックリストで進捗管理

GFM (GitHub Flavored Markdown) のタスクリスト構文を使うと、
サイドパネルにチェックボックスが表示されます。

```markdown
## サブタスク

- [ ] HTML の基本要素を理解した
- [ ] セマンティックタグを使い分けられる
- [x] 環境構築が完了した   ← チェック済みの初期値
```

## 保存の仕組み

チェック状態はブラウザの **localStorage** に自動保存されます。

```
localStorage キー: roadmapper:progress
構造: {
  "[roadmapId]": {
    "[nodeId]": {
      "state": "none" | "in-progress" | "done" | "skipped",
      "tasks": [true, false, true, ...]
    }
  }
}
```

ページを閉じても状態が保持され、再度開いたときに復元されます。

## ノード全体のステータス

サイドパネル上部のドロップダウンでノード全体のステータスを手動設定できます。

| ステータス | 意味 |
|---|---|
| `none` | 未着手 (デフォルト) |
| `in-progress` | 学習中 |
| `done` | 完了 |
| `skipped` | スキップ |
